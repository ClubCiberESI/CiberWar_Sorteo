        El algoritmo de reparto aleatorio selecciona un único participante no asignado y no manual cada vez que se pulsa el logo "VS". La decisión de asignar al equipo rojo o azul depende de los tamaños actuales: el equipo con menos miembros tiene más probabilidad de recibir al siguiente, y nunca se permite que la diferencia de tamaños supere uno. Las asignaciones manuales marcan al participante como excluido del reparto automático y se preservan incluso tras recargar o reiniciar, de modo que no vuelven a entrar en la cola de sorteo.

        De este modo se mantiene equidad dinámica (balance estrictamente 0‑1) y total transparencia sobre cómo se determina el resultado.

        function chooseTeamWeighted() {
    const redCount = participants.filter(p => p.team === 'red').length;
    const blueCount = participants.filter(p => p.team === 'blue').length;
    // si no hay asignados, moneda al aire
    if (redCount + blueCount === 0) return Math.random() < 0.5 ? 'red' : 'blue';
    // evitar diferencia mayor que 1
    if (redCount > blueCount + 1) return 'blue';
    if (blueCount > redCount + 1) return 'red';
    const probRed = blueCount / (redCount + blueCount);
    return Math.random() < probRed ? 'red' : 'blue';
}
